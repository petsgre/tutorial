# 设计模式

设计模式就是一套经过前人总结，并且不断验证实践过的一套代码组织方式

### [设计模式六大原则](http://www.uml.org.cn/sjms/201211023.asp)

1. 单一职责原则
2. 里氏替换原则
3. 依赖倒置原则
4. 接口隔离原则
5. 迪米特法则
6. 开闭原则

### 总结

经过一段时间的设计模式学习，发现很多情况下，已经在运用了，只是不知道他的名称概念，并且有些设计模式仅有细微的差别，或者说很相似。干巴巴的学习设计模式其实并没有什么用，重要的还是多写代码，多总结，才能写出更好的代码。
